Grocery Store Management System

This project is an SQL-based database system to manage grocery store operations. It handles data for products, suppliers, customers, employees, and orders. The system helps analyze sales, find top-selling products, and identify loyal customers and revenue trends.

Main objectives include designing a relational database, retrieving and analyzing data using SQL queries, and improving business decisions through insights.

Key tables used are Suppliers, Products, Customers, Employees, Orders, Order_Details, and Categories.

Challenges faced include maintaining relationships between tables, using correct joins, ensuring data consistency, and handling large datasets.
